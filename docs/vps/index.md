## 初遇

回忆往昔，第一次登录服务器是在毕业后的第一家公司（之前接触过的只有宝塔），当时的运维老哥给了我一个测试服务器的账号，还给我说了一款界面化的软件`finalshell`，当时登录上看着满是命令行的窗口，瞬间感觉头大，最后只不过是通过`finalshell`上传了打包后的前端文件到服务器上。至于为什么要前端打包，我记得当时是要做一个什么测试，然后这个项目的`ci/cd`还没有搭建。
这就是我第一次真实接触到服务器的经历，当时只感觉它是一个黑盒子，里面的东西我完全不知道。上学时，自己经常听到`云`这个名词，自己也参与过一些腾讯云的学生优惠活动，只不过当时只是用了`云数据库`，并没有接触过服务器。但是这次也算接触也算搞懂了一个概念，其实服务器就是一个`电脑`。

## 相识

拥有一台自己的服务器是在 2023 年 11 月，机缘巧合之下看到了阿里云的一个活动-`99元/年 2核2G 1M带宽`，当时也没想买来干嘛，就是感觉便宜就买了,可能这就是广告学的魅力吧。买来之后闲置了一段时间，直到有一天逛论坛看到一个帖子，说可以利用闲置服务器薅羊毛就点击进去了，该帖子是介绍使用青龙面板领取京东的京豆，当时跟着教程也在服务器上学着搭建面板。看别人介绍的很简单，实际自己上手还是层层遇阻。首先就是 linux 系统中的编辑器`vim`，第一次接触是真不会用啊(现在用的也不熟练)！进入之后，修改文件后怎么退出成为了我成功路上的第一个“麻烦”，因为总是忘记怎么切换进入底部命令模式。其次就是`docker`,这个平时在 windows 客户端操作操作还可以，在服务器上使用命令行操作还真是*一步查一下*，因为阿里云没有办法直接访问 docker 官方源，只能找网上的镜像源。最后通过一次次的尝试更换`镜像源`,还是成功下载下来了青龙面板的镜像。一套操作下来浏览器界面打开了得 20 个，索性还是成功运行了，也算自己在服务器上部署的第一个项目，也意识到了 docker 的好处（帮助用户解决环境依赖搭建的问题，这也为我后来使用 docker 部署前后端项目埋下了祸根）
